## 目标与范围
- 将站点部署到你当前电脑的本机 IP，以便同一局域网内的其他设备通过 `http://<你的IP>:<端口>` 访问。
- 先实现局域网可访问；如需公网访问，再进行端口映射/隧道配置。

## 环境与假设
- 操作系统：macOS（你的当前环境）。
- 项目类型未知，方案覆盖：静态前端构建产物（Vite/React/Vue 等）与 Node/Next.js 服务端渲染。
- 默认端口：`8080`（静态）、`3000`（Next/Node），可按需调整。

## 总体步骤
1. 识别项目类型与构建方式（静态构建产物 vs 运行时服务）。
2. 产出可部署的构建结果（如 `dist/`、`.next/`）。
3. 在本机以 `0.0.0.0` 绑定方式启动服务，监听指定端口。
4. 获取本机局域网 IP 并在其他设备通过 IP+端口访问。
5. 处理防火墙与网络连通性，确保入站连接被允许。
6. 验证跨设备访问，必要时做路径、代理或端口调整。

## 静态站点部署（如 Vite/React/Vue）
1. 构建：`npm run build`（或项目自带构建命令），得到 `dist/`。
2. 启动静态服务（无需安装额外软件，首选 Python）：
   - `cd dist`
   - `python3 -m http.server 8080 --bind 0.0.0.0`
3. 访问地址：`http://<你的局域网IP>:8080/`。
4. 如需更专业：后续可用 Nginx/Caddy 提供更完善的静态托管与缓存。

## Next.js/Node 服务部署
- Next.js：
  1. `npm run build`
  2. `npm run start` 等价于 `next start -H 0.0.0.0 -p 3000`
  3. 访问：`http://<你的局域网IP>:3000/`
- Express/Koa/自定义 Node 服务：
  1. 确认 `app.listen(PORT, '0.0.0.0')` 或 CLI 以 `--host 0.0.0.0` 启动。
  2. 访问：`http://<你的局域网IP>:<PORT>/`

## 获取本机 IP
- Wi‑Fi：`ipconfig getifaddr en0`
- 有线：`ipconfig getifaddr en1`
- 或在“系统设置 → 网络”查看当前连接的 IP。

## 防火墙与网络连通
- 确保本机与访问设备在同一局域网/Wi‑Fi，不在隔离网络/VPN 下。
- macOS 会弹窗询问是否允许 Python/Node 接受入站连接，选择“允许”。
- 若未弹窗：系统设置 → 隐私与安全 → 防火墙 → 为启动的程序允许入站连接。
- 路由器若启用客户端隔离（AP Isolation），需关闭以允许设备互访。

## 可选：公网访问（非必须）
- 家庭网络公网访问需在路由器做端口映射（WAN→LAN），并使用公网 IP：
  - 将外网端口（如 80/443 或自定义）映射到本机对应端口。
  - ISP 若提供动态 IP，建议使用 DDNS 或 Cloudflare Tunnel 等免端口映射方案。
