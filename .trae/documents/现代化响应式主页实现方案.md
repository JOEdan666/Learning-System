## 总体选型
- 框架：基于现有 Next.js 14 App Router（保持与项目一致）
- 样式：沿用 Tailwind CSS + CSS 变量（主题），组件内可使用少量 CSS Modules（复杂布局）
- 动画：Framer Motion（进场/过渡），配合 Tailwind 的 transition
- 主题切换：next-themes（`light`/`dark`），通过 CSS 变量驱动颜色
- 图标与插画：Lucide/自定义 SVG，避免体积膨胀

## 页面结构
- 顶部导航栏（Header）
  - 品牌 Logo（“天·才”）
  - 主导航：主页 / 系统学习 / 学习记录 / 知识库 / 统一对话
  - 用户入口：登录/注册、主题切换、语言切换（可选）
  - 响应式：移动端汉堡菜单 + 抽屉；桌面端水平导航
- 主体内容（Main）
  - Hero（首屏）：标题、副标题、主CTA（开始系统学习）、次CTA（试用演示），右侧产品截图/演示
  - 卖点三联（Benefits）：系统讲解 / 高效测验 / 间隔复习
  - 交互演示（Interactive Demo）：选择学科/年级/地区→生成一段讲解（懒加载）
  - 社会证明（Social Proof）：案例数据+评价卡片
  - 功能导览（Features Grid）：学习流程、知识库、统一对话、错题集、学习历史
  - 转化区域（Pricing/CTA）：免费试用说明与注册入口
- 页脚（Footer）
  - 版权、备案/隐私、帮助中心、联系与社媒链接

## 组件清单
- LayoutShell（统一骨架）
- HeaderNav（导航含主题切换）
- HeroSection（首屏）
- BenefitCards（三联卡）
- DemoPlayground（交互演示，懒加载）
- ProofSection（社会证明）
- FeaturesGrid（功能导览）
- ConversionStrip（转化条）
- FooterBar（页脚）

## 样式与主题
- Tailwind + CSS 变量：定义 `--bg`, `--fg`, `--primary`, `--muted` 等颜色；`dark` 模式下覆盖变量
- 组件层次：统一圆角/阴影/间距；Markdown 输出使用统一的 `.markdown-body` 主题
- 可访问性：对比度≥4.5:1，焦点态可见，语义标签（header/main/footer/nav）

## 响应式设计
- 断点：`sm/md/lg/xl`；首屏栅格布局（图文左右），在 `sm` 下堆叠
- 导航：移动端抽屉；组件网格在中小屏自动 1-2 列
- 图片：`next/image` 自适配，`sizes` 指定不同断点宽度

## 动画与过渡
- 页面过渡：`framer-motion` 的 `AnimatePresence`，首屏组件淡入+轻微位移
- 交互演示：生成卡片的渐显；按钮悬停动效克制（不影响性能）

## 性能优化
- 代码分割：大组件（DemoPlayground、ProofSection）`dynamic import({ ssr:false })`
- 懒加载：首屏以外模块在视口接近时加载（`IntersectionObserver`）
- 资源优化：`next/image`，SVG inline；减少第三方依赖体积
- 预取与缓存：重要路由 `prefetch`；接口使用 SSG/ISR 或客户端缓存（轻演示走客户端）

## SEO与结构化数据
- `app/layout.tsx` 使用 Metadata：title、description、openGraph、twitter
- JSON-LD：Organization + WebSite + Breadcrumb（主页）
- 语义化标签与标题层级（H1/H2）

## 访问统计
- 集成 `@vercel/analytics` 或 Plausible/GA（轻量优先）
- 事件：首屏 CTA 点击、演示完成、注册点击

## 主题切换
- `next-themes` 提供切换与系统偏好；持久化在 `localStorage`
- 切换时只变更 CSS 变量（避免重算大量样式）

- 组件文档：在 `app/components` 注释 + 说明，提供使用示例
- 兼容性测试：Chrome/Firefox/Safari/Edge；移动端 iOS/Android 近两代
- Lighthouse：性能≥90、可访问性≥90、最佳实践≥90、SEO≥90
